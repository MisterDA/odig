Updating GitHub pages

type update = B0_std.Fpath.t * B0_std.Fpath.t option

The type for updates.

  • (p, None) deletes path p in the work tree.
  • (p, Some r) replaces the relative path p in the work tree by the contents of r (which may be a file hierarchy).

In all cases p must be a relative path and may not exist in the work tree in which case it is created. Use . for p to denote the root of the work tree; for example Fpath.v ".", Some dir replaces the whole website by the file hierarchy rooted at dir.

Warning. If you have paths that start with _ GitHub pages ignores. You can disable this by adding a `.nojekyll` file at the root the gh-page branch.

val default_branch : B0_vcs.Git.branch

default_branch is "gh-pages", GitHub's default publication branch for GitHub pages.

val update : ?⁠log:B0_std.Log.level -> ?⁠branch:B0_vcs.Git.branch -> B0_vcs.t -> force:bool -> remote:B0_vcs.Git.remote -> msg:string -> update list -> (bool, string) Pervasives.result

replace_dirs vcs ~log ~remote ~branch ~msg us updates branch (defaults to gh-pages) on remote according to us (see Updating GitHub pages) with commit message msg. If force, the various git operations are forced. log indicates a logging level used to monitor progress (defaults to Log.app). Ok false is returned if there was nothing to update.

More precisely this:

  1. Fetches remote/branch if it exists.
  2. Creates a transient checkout with a temporary workdir in Os.Dir.default_tmp and a branch called _b0-update-gh-pages reset to remote/branch.
  3. Commits changes with message msg according to us which are applied in order, see Updating GitHub pages.
  4. Destroys the transient checkout
  5. Pushes _b0-update-gh-pages on remote/branch
  6. Destroys the branch _b0-update-gh-pages
