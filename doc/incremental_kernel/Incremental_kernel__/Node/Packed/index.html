Module Node.Packed

For performance reasons, we do not use an OCaml existential type for Node.Packed.t:

type t = T : _ Node.t -> t

The extra indirection when following pointers to packed nodes would be too slow.

Consequently, there is a possible bug in which we mix the 'a from two packed nodes with different types. We reduce the chance of this bug by minimizing the scopes in which we deal with packed nodes.
