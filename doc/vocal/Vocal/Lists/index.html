Stack-safe and fast implementation of List.map

following antron's post https://discuss.ocaml.org/t/a-new-list-map-that-is-both-stack-safe-and-fast/865

Important note: the function is applied to the elements starting from the end of the list, thus not in the same order as with List.map. So if your functions has side-effects, this is not equivalent to List.map but rather to a combination of List.rev and List.map as stated below.
